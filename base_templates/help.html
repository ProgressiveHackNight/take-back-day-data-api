Help

There are several ways to work with the interface:

  1. Import data to the server (either as JSON, or as an .xlsx spreadsheet, or both)
  2. Export the data to JSON for use in the public web site.

When new data, or updated data is available, either start a new iteration of the same process, or edit the database directly on the server using the Django admin interface.

  • Use the Django admin interface to add, delete, and edit records in the database, or
  • Export to a format that is suitable for further editing, such another JSON file, or an .xlsx file for use in a spreadsheet.

Getting data from the database

JSON Data exported from the database is available at https://takebackday.infoloom.com/media/output.json

Updating data

Workflow 1. Working with a spreadsheet

  • Empty the database.
  • Import data from spreadsheet.
  • Export to JSON

Repeat this process when spreadsheet is updated.

Note. No "upload" function is implemented yet. For now, just send the file as an email attachment to mb@infoloom.com

Workflow 2. Working with diverse data

  • Empty the database.
  • Import data from spreadsheet, and various JSON files obtained from scraping.
  • Export to JSON
  • Export combined data to spreadsheet, for next iteration.

Note. Data structure should be consistent to use the predefined imports. If the data structure changes, new import functions must be created to accommodate new data.

Workflow 3. Import once, use the admin to edit the database.

Import multiple data sets. They will be incorporated into the database.

  • Use the admin interface to edit the data in the database.
  • Export when needed to JSON.

In this case, do not delete the database.

Note. If this solution is adopted, we will remove the ability to delete the database (and should probably backup the database on the server regularly).
